Отличия машинного обучения и классического программирования¶

Типичный подход к решению задач в программировании состоит из следующих операций:

  • поиск правил, которые нужно применить ко входным данным, чтобы получить правильный результат. Набор таких правил называется алгоритмом

  • описание алгоритма на языке программирования

  • тестирование на входных данных, похожих на реальные, коррекция алгоритма\имплементации. Обычно известен сценарий, в котором будет применяться алгоритм,и для тестирования корректности программы подготавливается набор тестовых данных в виде "входные данные" - "правильный ответ". Запомните эту идею, она имеет фатальное значение для понимания коцепции всего, что будет описано далее.

Однако существуют задачи, в которых вывести правила методом анализа условий задачи не получается. Это может быть связано с тем, что правила, по которым данные возникают не известны никому, либо - проблема слишком сложна для анализа в принципе.

Машинное обучение - это компьютерная наука, изучающая задачи, для решения которых нет известных правил, однако существуют массивы примеров реальных данных. Учёные этого направления создают подходы к решению задач, которые можно разделить на несколько основных направлений:

При решении задач методами машинного обучения, специалисты не создают алгоритм, а формулируют задачу как разновидность обучения. После этого выбирают модель, подготавливают данные и прогоняют модель в режиме обучения. Процесс повторяют до тех пор, пока алгоритм не сойдется до нужного\возможного значения точности.

def PerceptronANDInference(A, B):
    b = -50
    w1 = 30
    w2 = 30
    x1 = A
    x2 = B
    return sigmoid(w1*x1 + w2*x2 + b)
